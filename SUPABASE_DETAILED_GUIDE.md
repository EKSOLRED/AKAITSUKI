# AKAITSUKI + Supabase

## Что уже настроено в архиве

В проекте уже заполнены:

- `js/config/supabase.config.js`
- `js/config/app.config.js`

То есть авторизация через Supabase уже включена.

Текущие значения:
- URL: `https://gvxbeglgslgnnxhuihwe.supabase.co`
- Key: publishable key подключён в клиент

## Что нужно сделать в панели Supabase

### 1. Включить email/password auth

Открой:
**Authentication → Sign In / Providers**

Включи Email provider.

### 2. Настроить URL Configuration

Открой:
**Authentication → URL Configuration**

Для локальной разработки добавь:

- Site URL: `http://localhost:5500`
- Redirect URL: `http://localhost:5500/`

Для продакшена потом добавь:
- `https://your-domain.com/`

### 3. Создать таблицу профилей

В SQL Editor запусти:

1. `supabase/004_profiles_schema.sql`
2. `supabase/005_profiles_rls.sql`

Это создаст:
- enum ролей
- таблицу `public.profiles`
- RLS политики

## Как работает логика ролей

После логина сайт читает роль из `public.profiles.role`.

Поддерживаются роли:
- `user`
- `admin`
- `owner`

### Сделать первого владельца

1. Зарегистрируйся через сайт
2. Выполни `supabase/006_make_owner.sql`
3. Замени email на свой

## Как проверить, что всё работает

### Регистрация
- открой сайт
- зарегистрируй аккаунт
- проверь, что в `Authentication → Users` появился пользователь
- проверь, что в `Table Editor → profiles` появилась строка

### Логин
- выйди
- снова зайди
- если роль `owner`, должна открываться админка

### Восстановление пароля
- в окне логина нажми `Забыли пароль?`
- введи email
- Supabase отправит письмо на reset

## Что пока не переведено полностью в Supabase

В этом архиве авторизация уже на Supabase, но основной контент сайта пока работает в локальном режиме:

- каталог
- тайтлы
- избранное
- about-контент
- рейтинги

Для этого `dataMode` всё ещё стоит в `local`.

То есть:
- Auth уже боевой
- Контент пока локальный

## Как безопасно выкатывать обновления

### Перед правками
1. экспортируй SQL из изменяемых таблиц
2. сохраняй копию архива проекта
3. новые SQL изменения складывай отдельными файлами:
   - `007_...sql`
   - `008_...sql`

### Откат
Если сломал SQL-обновление:
- либо выполняй обратный SQL вручную
- либо восстанавливай данные из backup

### Для фронтенда
- храни рабочие архивы по версиям:
  - `release-0.9.1.zip`
  - `release-0.9.2.zip`

## Перевод контента на Supabase

Теперь проект работает в `dataMode: 'supabase'`.

Это значит:
- тайтлы читаются из `public.titles`
- избранное читается из `public.favorites`
- рейтинги читаются из `public.title_ratings`
- контент `О нас` читается из `public.site_content`

### Автоматический первичный перенос
Если таблицы пустые и ты вошёл как `admin` или `owner`, сайт попробует один раз перенести текущие локальные тайтлы и блок `О нас` в Supabase.

### Что не перенесётся автоматически
- старые локальные пользователи
- старые локальные пароли
- локальные избранные и рейтинги других пользователей

Это нормально, потому что теперь источником истины становится Supabase.
