# CSS Phase 3

Этот проход делает структуру проекта ещё чище, но сохраняет тот же каскад.

Что изменено:
- большие агрегаты разбиты на подпапки по исходным этапам фиксов;
- `pages/favorites.css`, `pages/admin.css`, `overrides/legacy-fixes.css`, `overrides/ui-hotfixes.css` теперь работают как индекс-файлы через `@import`;
- порядок импортов сохранён, чтобы не менять визуал.

Новая структура:
- `styles/pages/favorites/` — история фиксов страницы избранного;
- `styles/pages/admin/` — фиксы админки;
- `styles/overrides/legacy/` — старые поэтапные фиксы layout/detail/UI;
- `styles/overrides/ui/` — поздние UI hotfix-файлы.

Это не агрессивная оптимизация, а безопасная декомпозиция. Следующий шаг уже можно делать точечно: переносить стабильные правила из hotfix-файлов в нормальные component/page файлы и удалять дубли после визуальной проверки.
